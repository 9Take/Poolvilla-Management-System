🚀 Deployment & Configuration

Docker Compose Stack (Container 1)

version: '3.8'

services:

  n8n:
    image: n8n:latest
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=secure_password
    volumes:
      - n8n_data:/home/node/.n8n
    networks:
      - oms_network

  fastapi:
    build:
      context: ./fastapi
    ports:
      - "8000:8000"
    depends_on:
      - postgres
      - redis
    networks:
      - oms_network

  postgres:
    image: postgres:15-alpine
    environment:
      - POSTGRES_DB=oms_db
      - POSTGRES_USER=oms_user
      - POSTGRES_PASSWORD=secure_password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - oms_network

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - oms_network

volumes:
  n8n_data:
  postgres_data:
  redis_data:

networks:
  oms_network:
    driver: bridge

Startup Commands

# Create LXC Container (from Proxmox CLI)
pct create 100 local:vztmpl/ubuntu-24.04-standard_24.04-1_amd64.tar.zst

# Access container
pct shell 100

# Install Docker & Docker Compose
apt update && apt install docker.io docker-compose -y

# Start services
docker-compose -f docker-compose.yml up -d

# Check status
docker-compose ps
docker logs -f n8n # View n8n logs

💾 Data Persistence & Backup Strategy

Primary Storage

Location: M.2 NVMe (128GB)
Purpose: OS + Docker containers
Speed: High-speed R/W operations

Data Storage

Location: SATA SSD (500GB)
Purpose: PostgreSQL data files
Speed: Consistent performance

Backup Strategy

Database: Daily automated backups
Configuration: Git versioning
Volumes: Scheduled snapshots
Retention: 30-day rolling window

Disaster Recovery

RTO: 30 minutes
RPO: 1 hour
External: Cloud storage backup
Testing: Monthly recovery drills

📈 Scaling & Performance Considerations

Current Capacity

• Concurrent Users: ~50-100
• Daily Requests: ~10,000
• Storage: ~50GB (current)
• Response Time: <500ms

Bottlenecks

• CPU: n8n workflow execution
• Memory: Redis cache during peak
• Disk I/O: PostgreSQL writes
• Network: Upstream API calls

Optimization Paths

• Vertical: Upgrade to NUC with i7
• Horizontal: Multi-node Proxmox cluster
• Database: PostgreSQL replication
• Caching: Advanced Redis patterns

Monitoring Metrics

• CPU Usage: Alert >80%
• Memory: Alert >85%
• Disk: Alert >90%
• Response Time: Track p95 latency

🔒 Security Architecture

Security Layers

Layer 1: Network Security
• Proxmox firewall rules
• UFW (Ubuntu Uncomplicated Firewall)
• Port mapping & isolation

Layer 2: Application Security
• JWT authentication
• Input validation & sanitization
• Rate limiting & DDoS protection

Layer 3: Data Security
• Database encryption (TLS 1.3)
• Password hashing (bcrypt)
• Secrets management (Docker secrets)

Layer 4: Operational Security
• Regular security audits
• Log monitoring & alerting
• Penetration testing quarterly
